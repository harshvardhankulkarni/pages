Project Overview

Chemsol is a comprehensive Zoho Creator enterprise application purpose-built for industrial flooring solutions — epoxy, polyurethane, anti-static, ESD, demarcation, and coving systems. The application manages the complete product lifecycle: procurement of raw materials, quality control, production batch manufacturing, store/inventory management, project execution, and sales order processing.

Business Context

🎯 Product Range

EP (Epoxy), PU (Polyurethane), DEM (Demarcation), NUM (Numbering), ARR (Arrow Marking), ANTI (Anti-static), ESD (Electrostatic Discharge), FIL (Filling), COV (Coving) — each with unique system codes.

⚡ Execution Modes

Projects operate on Area-basis (sq. ft. of flooring installed) or Day-basis (manpower days). Sales/Work Orders support Supply+Apply and Supply-only models.

🏢 Department Coverage

Admin, Purchase, Store, Production, QC, Sales, Project Management, and Finance — each with dedicated forms, reports, and role-based permissions.

🔒 Security Model

13 user roles with form-level C/R/U/D, field-level visibility restrictions, record-level sharing rules, and approval limit thresholds (₹50K PR, ₹2L PO).

// Dual auto-number (On Create — based on RM Type)
if(input.RM_Type == "Coding") { prefix = "RMWAD"; }
else { prefix = "RM"; }
lastRecord = zoho.creator.getRecords("chemsol", "PO_Master",
    "PO_Number.startsWith('" + prefix + "')", 1, 0);
if(lastRecord.size() > 0) {
    lastNum = lastRecord.get(0).get("PO_Number").split("-").get(1).toLong();
    newNum = lastNum + 1;
} else { newNum = 1; }
input.PO_Number = prefix + "-" + newNum.toString().padLeft(3, "0");

// GST Calculation per line item (On Qty/Rate/GST% change)
basicAmount = input.Qty * input.Rate;
gstAmount = basicAmount * input.GST_Percentage / 100;
if(input.GST_Type == "Intra-State") {
    input.CGST_Amount = gstAmount / 2;
    input.SGST_Amount = gstAmount / 2;
} else {
    input.IGST_Amount = gstAmount;
}
input.Total_Amount = basicAmount + gstAmount;

5Goods Receipt NoteGRN_Master

Full GRN with PO reference auto-fetch, partial receipt (checkbox per line), QC Status tracking, Vehicle/Transport details. Post-GRN button generates GRN number, updates stock (Item_Master.Current_Stock += Received_Qty), updates PO status.

// Post GRN Logic
if(input.Status == "Draft") {
    // Validate
    selectedItems = input.GRN_Items.where(Select == true);
    if(selectedItems.size() == 0) { throw "Select at least one item"; }
    for each item in selectedItems {
        if(item.Received_Qty <= 0 || item.Received_Qty > item.Balance_Qty) {
            throw "Invalid quantity for " + item.Item_Name;
        }
    }
    // Generate GRN Number
    input.GRN_Number = "GRN-" + getNextSequence("GRN_Series");
    // Update Stock
    for each item in selectedItems {
        itemRecord = zoho.creator.getRecordById("chemsol", "Item_Master", item.Item_Code);
        currentStock = itemRecord.get("Current_Stock");
        zoho.creator.updateRecord("chemsol", "Item_Master",
            item.Item_Code, {"Current_Stock": currentStock + item.Received_Qty});
    }
    // Update PO Received Qty
    // Set Status = "Posted", Posted_Timestamp = now
    input.Status = "Posted";
    input.Posted_Timestamp = zoho.currenttime;
}

Testing Strategy

Test Approach: Each phase has a dedicated testing plan covering field validation, workflow execution, data integrity, and cross-form integration. UAT scenarios are documented per department.

🧪 Unit Testing

Per-form field validation: required fields, data types, regex patterns (HSN, GST), unique constraints, auto-numbering sequences.

🔗 Integration Testing

Cross-form data flow: PR→RC→PO→GRN→QC, MR→MIS→Stock, BOM→Production→Packing. Verify auto-fetch lookups and stock updates.

🔐 Security Testing

Role-based access: verify form-level C/R/U/D per matrix, field-level restrictions (Standard Rate, Bank Details), approval limit enforcement.

📊 Report Testing

Data accuracy: verify summary groups, date range filters, calculated fields, rollup totals. Print format validation for PO, GRN, QC reports.

⚡ Performance Testing

Form load times, lookup response, report generation with 200+ records. Deluge function timeout (5-min limit) for batch operations.

📱 Mobile Testing

Form rendering on Zoho Creator mobile app, responsive layouts, offline data entry sync for field workers (Project tasks, MR/MIS).

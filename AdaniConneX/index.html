Overview

AdaniConneX is implementing ZohoCRM Enterprise to automate Data Center Opportunity sales workflows across lead management, sales cycle, contracting, and capacity management. This plan covers the full lifecycle from opportunity creation through SOF Contracted (100%), including cross-team approvals, pricing validation, document management, dependent system integrations, and contract renewals.

Team Responsibilities

Sales

Creates and maintains Opportunities, Accounts, Contacts, Calls, and Meetings. Drives deals up to SOF MSA.

Sales Operations

Validates pricing, drives data accuracy, approves SOF MSA stage movement. Edits post-SOF-Contracted.

Commercials

Validates commercial details. Approval required on SOF MSA (second in sequence).

Pre-Sales

Validates technical feasibility. Approval required on SOF MSA (first in sequence).

IT

Administers the system. Full access to all modules and configurations.

Service Delivery

Receives order notifications, initiates billing in SAP, sets Deployed Date.

3 Approval Workflow — SOF MSA 3–4 days

Sequential approval process when an opportunity moves to SOF MSA. Pre-Sales (technical feasibility) approves first, then Commercials (commercial validation).

Sales moves to SOF MSA
     │
     ▼
Pre-Sales Approval Request (Technical Feasibility)
     │
     ├── Approved → Commercials Approval Request
     │                 │
     │                 ├── Approved → Stage moved to SOF MSA
     │                 │                     → Notification sent
     │                 │
     │                 └── Rejected → Returned to Negotiation
     │
     └── Rejected → Returned to Negotiation

ZohoCRM Implementation: Blueprints handle stage transitions. Two Approval Processes: APPROVAL_SOFMSA_PRESALES (triggered first) and APPROVAL_SOFMSA_COMMERCIAL (triggered only on Pre-Sales approval). 48-hour escalation to reporting manager if no response.

Open: Confirm 48-hour escalation SLA, fallback approvers, and whether approvals go to individuals or teams per site/region.

7 Dependent System Integrations 10–15 days

7A. EdgeOS Portal

Push data from ZohoCRM to EdgeOS every 15 minutes for SOF Contracted opportunities: Opportunity Reference ID, Account Name, Site ID, Contracted Power (KW), Contract Status, and Contract Expiry Date.

Architecture Options: (A) Custom Function → Webhook → EdgeOS API (recommended if EdgeOS has REST API). (B) Scheduler + Deluge → JSON export → FTP/SFTP (fallback). (C) Middleware (Zapier/Workato).

7B. SAP Billing

Triggered when Service Delivery updates Deployed Date. Exports: Opportunity Reference ID, Account + billing address, product lines, MRC, OTC, contract start date and term.

Architecture Options: (A) Custom Function → SAP API (direct). (B) Flat file (CSV) → FTP/SFTP (recommended initially). (C) Middleware (Boomi, MuleSoft).

Open: EdgeOS API availability and auth mechanism. SAP import file format (CSV/IDoc/API). Existing middleware availability. EdgeOS sync scope — only SOF Contracted or also SOF MSA?
8 Contract Management & Renewals 2–3 days

Contract Expiry Date auto-calculated via formula: Deployed Date + Contract Term (Months). Renewals captured via Renewed Through Date (manual entry). Expired contracts set to SOF Expired stage; early terminations set to SOF Cancelled.

Open: Expiry notification lead time. Does renewal update existing opportunity or create new? EdgeOS behaviour on contract expiry (revoke portal access?).

Customer feedback collection via QR codes at data center sites and email scheduler. Aggregates ratings from Sales, Service Delivery, Service Assurance, Security, Billing, Customer Experience, and Overall Organization.

11 Data Migration & Go-Live 3–5 days + Hyper-care

Pre-Migration Checklist

  • All modules, fields, picklists configured and tested
  • EDC Products catalog populated (~50 products)
  • User roles, profiles, and permissions created and assigned
  • Blueprints and approval processes tested end-to-end
  • Document upload flows verified
  • Notifications sending to correct recipients
  • EdgeOS integration tested
  • SAP export format validated with SAP team
  • Reports and dashboards created

Go-Live Sequence

1. Cutover: Freeze legacy system input
2. Import Accounts & Contacts
3. Import Opportunities (open pipeline)
4. Configure EdgeOS initial sync
5. User training sessions (all teams)
6. Go-Live: ZohoCRM becomes system of record
7. Hyper-care (2 weeks): IT + Sales Ops monitor
8. Post-go-live: Configure scheduler

Open Decisions

These 12 decisions must be resolved before the corresponding implementation phases can proceed.

  • #1
    SiteID values list — Define the picklist values (e.g., EDCPOR01, BLR01, MUM01). Owner: Sales Ops / IT. Blocks: Auto-number setup.
  • #2
    6-digit counter scope — Global sequence or per-SiteID counter? Owner: Sales Ops / IT. Blocks: Auto-number setup.
  • #3
    Deployed Date ownership — Who sets this field? Service Delivery or Sales Ops? Blocks: Contract expiry auto-calculation.
  • #4
    Expiry notification interval — Single reminder or staged (90d + 30d + 7d)? Owner: Sales Ops. Blocks: Scheduler config.
  • #5
    CSAT Dashboard — In-scope (Zoho) or separate external system? Owner: Stakeholder decision.
  • #6
    EdgeOS API — Is a REST API available? What is the auth mechanism? Owner: EdgeOS Team. Blocks: Integration design.
  • #7
    SAP import format — CSV, IDoc, or API? Owner: SAP Team. Blocks: Integration design.
  • #8
    Approval escalation SLA — 48 hours or different? Fallback approvers? Owner: Sales Ops / Pre-Sales / Commercials.
  • #9
    Standard price list — Rate card for validation, or fully negotiated per deal? Owner: Sales Ops / Commercials.
  • #10
    Combined PO+MSA document — Can a single document satisfy both requirements? Owner: Sales / Legal.
  • #11
    Terminal transitions — Which stages can go to Lost/Dropped/Material Change? Owner: Sales / Sales Ops.
  • #12
    Renewal approach — New opportunity or update existing record? Owner: Sales Ops.

## 3. Creator Implementation Steps

### 3.1 Create Form

1. Form > Create Form > name = "Item Master", internal name = `Item_Master`

### 3.2 Add Fields

1. **Category of Purchase Item**: Insert > Dropdown > Enter 5 options (RM, PM, FG, CON, SVC) > Check "Required"
2. **Purchase Item Code**: Insert > Text > Max 10 > Check "Required", "Unique", and "Read-only"
3. **Purchase Item Name**: Insert > Text > Max 200 > Check "Required"
4. **UOM**: Insert > Dropdown > Enter 13 options > Check "Required"
5. **HSN Code**: Insert > Text > Max 10 > Check "Required" > Properties > Custom Validation > Pattern: `^\d{4,8}$`
6. **GST %**: Insert > Decimal > 2 places > Do NOT check Required
7. **Min Stock**: Insert > Number > Uncheck Decimal > Check "Required" > Min Value = 0
8. **Max Stock**: Insert > Number > Uncheck Decimal > Do NOT check Required > Min Value = 0
9. **Standard Rate**: Insert > Currency > Currency = INR > Do NOT check Required
10. **Preferred Supplier**: Insert > Lookup > Target form: "Supplier Master" > Display field: "Supplier Name" > Enable "Autocomplete" > Do NOT check Required
11. **Lead Time (Days)**: Insert > Number > Uncheck Decimal > Do NOT check Required > Min Value = 0

### 3.3 Add Auto-Fetch Fields via Lookup

When adding the **Preferred Supplier** lookup field, Creator will show an "Add Lookup Fields" option:

12. Click "Add Lookup Fields" on the Preferred Supplier field
13. Select **Supplier Name** from the Supplier Master field list
14. Creator auto-creates a read-only field: **Supplier Name** (internal: `Supplier_Name`)
15. This field will auto-populate when a supplier is selected — no Deluge code needed

### 3.4 Create Sections

16. Form > Sections > Add Section: **Basic Details**, **Inventory & Purchase**
17. Drag fields into correct sections per layout (see §6)

### 3.5 Set Field-Level Security

18. Standard Rate > Properties > Access > Uncheck all profiles except Admin, Purchase Manager, Store Manager, Finance Executive
19. Finance Executive gets View only — uncheck Create/Update in Standard Rate's Access tab

## 4. Deluge Code — On Submit (Before) Workflow

### Trigger

- **Event**: On Submit
- **Timing**: Before record is added to database
- **Purpose**: Auto-generate Purchase Item Code + run validations

### Full Deluge Script

```deluge
// ============================================================
// Item Master — On Submit (Before)
// ============================================================
// 1. Auto-generate Purchase Item Code
// 2. Validate GST % range (0–100)
// 3. Validate Min Stock ≤ Max Stock
// 4. Validate duplicate Item Name
// ============================================================

// --- 1. Auto-generate Purchase Item Code ---
// Read the selected category prefix (RM, PM, FG, CON, SVC)
prefix = input.Category_of_Purchase_Item;

if(prefix != null && prefix != "")
{
    // Query the most recent record in this category by Purchase_Item_Code descending
    // This handles both manual imports (e.g., RM-005 already exists) and new entries
    last_records = zoho.creator.getRecords("chemsol", "Item_Master",
        "Category_of_Purchase_Item == '" + prefix + "'",
        1, "Purchase_Item_Code desc");

    last_seq = 0;

    if(last_records.size() > 0 && last_records.get(0).get("Purchase_Item_Code") != null)
    {
        last_code = last_records.get(0).get("Purchase_Item_Code").toString();

        // Parse the numeric suffix after the hyphen
        // Handles codes like "RM-005", "FG-012", "CON-100"
        if(last_code.contains("-"))
        {
            last_seq = last_code.substringAfter("-").toLong();
        }
    }

    // Increment and pad to 3 digits
    new_seq = last_seq + 1;
    formatted_seq = new_seq.toString().padLeft(3, "0");
    input.Purchase_Item_Code = prefix + "-" + formatted_seq;
}
else
{
    throw "Please select a Category before saving.";
}

// --- 2. GST % range check (0 to 100) ---
if(input.GST_Percent != null)
{
    if(input.GST_Percent < 0 || input.GST_Percent > 100)
    {
        throw "GST % must be between 0 and 100.";
    }
}

// --- 3. Min Stock ≤ Max Stock validation ---
// Only validate if both fields have values (Max Stock is optional)
if(input.Min_Stock != null && input.Max_Stock != null)
{
    if(input.Min_Stock > input.Max_Stock)
    {
        throw "Min Stock cannot be greater than Max Stock.";
    }
}

// --- 4. Duplicate Item Name check ---
if(input.Purchase_Item_Name != null && input.Purchase_Item_Name != "")
{
    // Use q() to escape special characters in the name
    criteria = "Purchase_Item_Name == '" + q(input.Purchase_Item_Name) + "'";
    // Exclude current record on edit to avoid false-positive
    if(input.ID != null)
    {
        criteria += " && ID != '" + input.ID + "'";
    }
    existing = zoho.creator.getRecords("chemsol", "Item_Master", criteria, 1, 0);

    if(existing.size() > 0)
    {
        throw "An item with this name already exists. Please use a different name.";
    }
}
```

### Code Sequence Notes

| Order | Validation | Why This Order |
|-------|-----------|----------------|
| 1 | Category selected | Must have prefix to generate code |
| 2 | Code generation | Core function — run first |
| 3 | GST % range | Quick data integrity check |
| 4 | Min ≤ Max | Cross-field logical check |
| 5 | Duplicate name | Most expensive query — run last |

### Test Cases (Auto-Number)

| Scenario | Category | Existing Codes | Expected Result |
|----------|----------|---------------|-----------------|
| First RM item | RM | (none) | RM-001 |
| Next RM item | RM | RM-001 | RM-002 |
| After RM-009 | RM | RM-001 to RM-009 | RM-010 |
| After data import with gap | RM | RM-001, RM-005, RM-010 | RM-011 |
| First FG item | FG | (none) | FG-001 |
| First CON item | CON | (none) | CON-001 |
| No category selected | (blank) | — | Error thrown |

# Plan 02-04: FastAPI Service with LangChain Agent Integration

## Goal
Build the FastAPI HTTP service that exposes the LangChain agent as REST endpoints. Includes 6 endpoints (`/chat`, `/forecast`, `/anomalies`, `/projects`, `/weekly`, `/health`) with Langfuse tracing, guardrails, and async agent invocation.

## Threat Model

  <threat>

    <category>ASVS L1 ‚Äî DoS / Resource Exhaustion</category>

    <mitigation>Agent has `max_iterations=5` hard limit. FastAPI's default concurrency handles requests (uvicorn workers). No per-IP rate limiting in this phase ‚Äî add via middleware or API gateway in production. OpenAI `max_retries=2` and `timeout=60` prevent hung requests.</mitigation>
    <severity>MEDIUM</severity>
    <status>ACCEPTED</status>
  </threat>
  <threat>
    <id>T-11</id>
    <category>ASVS L1 ‚Äî Output Injection</category>
    <description>If the LLM output contains HTML/script tags, returning it directly in `/chat` response could enable XSS if consumed by a web frontend.</description>
    <mitigation>The `/chat` endpoint returns JSON (`{"response": str}`) ‚Äî response is a plain text string, not rendered HTML. No sanitization needed at this layer. If a web UI renders this response, it should use text-interpolation (not innerHTML).</mitigation>
    <severity>LOW</severity>
    <status>ACCEPTED</status>
  </threat>
  <threat>
    <id>T-12</id>

    <severity>MEDIUM</severity>
    <status>ACCEPTED</status>
  </threat>

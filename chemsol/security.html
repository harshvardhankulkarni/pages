Roles · Permissions · Access Control

Security & User Roles

Roles are scoped by Project — Project Managers/Coordinators see only their assigned projects. Like Zoho CRM role hierarchy with record-level access.

Role Architecture

  • Administrator (IT)
    • Admin Full system access
      • Purchase Entry / Review / Approve
      • Store & Logistics Entry / Review
      • Production & R&D Entry / Review
      • Project Manager 1/2/3 / Coordinator
      • Sales Entry / Review
      • QC Entry / Review
      • Account & Finance View only

Project-Level Access Control

Like Zoho CRM record-level sharing — users see only data tagged to their assigned projects.

  • Project Manager 1/2/3 — Full CRUD on assigned projects/tasks. View-only on procurement/inventory for their projects.
  • Project Coordinator — Create MR for assigned projects, view production and project status.
  • Sales — Create SO (auto-creates project). View all projects.
  • Purchase — See all projects' procurement data. No task management access.
  • Store — See all projects' inventory movements. No project finance access.

Approval Matrix

Defines PR/PO limits and approvers per department. Multi-level escalation. Handled via Zoho Creator's native approval features (Blueprint / Approval processes) — no custom form needed.

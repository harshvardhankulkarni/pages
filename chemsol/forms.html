Forms & Module Structure

Core Workflow: SO → Costing (RM check) → MR (Draft → Prod Verified → Costing Approved → Released) → MIS → Production → FGHM → FG Stock. Procurement (PR→PO→GRN→QC) runs when stock insufficient. Inventory manages RM + FG.

3-Layer Hierarchy: SO sells Systems → composed of FGs (System Composition) → each FG formulated from RMs (BOM). MR works at RM level only.

Backend rule (consumption resolution): Every consumption of RM against the Project — from FG production (BMR / RM Consumption) or Site Manager entries — increments Consumed Qty on the matching MR Allocation line (matched by Project ID + Item Code). Never a generic pool deduction for project-tagged material.

MR Approval workflow: Draft → Production verifies MR qty against SO system requirements → Costing approves total project cost → Released (available for MIS). Notification to Project Manager on each change.

⚠ 80% Consumption Alert — Per Allocated Material (KEY REQUIREMENT)

Trigger: For each RM line in the MR Material Allocation subform, when Consumed Qty ÷ Assigned Qty × 100 ≥ 80% AND the 80% Threshold Alert Flag is ON, raise a warning.

Example: Material X assigned 10 units → alert fires when X has been consumed 8 units (80%). Material Y (20) → at 16 units. Material Z (30) → at 24 units.

Notifications fired: (1) In-app pop-up to the user saving the consumption entry; (2) Dashboard banner on the Project dashboard; (3) Email to the Project Manager. Banner escalates to "Allocation Exhausted" at 100%.

6F Automation Rules & Notifications Business Rules

80% Consumption Alert

When any allocated RM in MR Material Allocation reaches ≥ 80% consumption (with Alert Flag ON) → pop-up + dashboard banner + email to Project Manager. Escalates to "Allocation Exhausted" at 100%.

Stock Management Rules

  • GRN posting: Qty added to stock only after posting; timestamp logged
  • MIS posting: Qty deducted from stock
  • FGHM inline acceptance: FG qty added to stock after inline acceptance
  • Material Return: Qty added back to stock
  • Min/Max stock: Alert when stock crosses thresholds
  • Consumption → Project-Assigned RM: Every consumption entry resolves to MR Allocation line matched by Project ID + Item Code — never a generic pool

Project-Centric ERP · Phases 1A→1L

Implementation Plan

Two-Stream Architecture across build phases. Stream A (Pre-Project): procurement for stock, no Project ID. Stream B (Project-Centric): Project is root entity for all downstream modules.

System Overview

Chemsol — Project-Centric ERP on Zoho Creator

Flooring & Construction Materials — Two-Stream Architecture

Two streams: Stream A (Pre-Project): Procurement runs independently for stock — PR→PO→GRN→QC→Material Handover→Store. No Project ID on procurement forms. Stream B (Project-Centric): Sales Order creates a Project. All modules — procurement, inventory, production, budget, billing — are tagged to that Project. Nothing exists outside a project context in Stream B.

Project-Centric Hub Architecture

Hub Architecture

Two-Stream Architecture: Stream A (Pre-Project) — PR, PO, GRN, QC carry no Project ID (stock procurement). Stream B (Project-Centric) — MR, MIS, Production, FGHM, Service, Finance, Logistics all carry Project ID lookup. Master Data (Item Muster, Suppliers, Customers, Store Master) is global.

Automation Rules System

Project-Centric Data Flow

Two-Stream Data Flow: Stream A (Pre-Project, no Project ID) runs independently. Stream B (Project-Centric) carries Project ID downstream.

Stream A: PR → Rate Comparison → PO → GRN → QC → Material Handover → Store

Stream B: SO (Supply+Apply) → Project → MR → MIS → Production (Plan → BMR → Pack → FGHM) → Costing → Service → Finance → Logistics

Every Stream B form inherits Project ID. Stream A forms carry no Project ID.

Stock Management

  • GRN posted → Qty added to stock (per warehouse). Timestamp logged.
  • MIS posted → Qty deducted from stock.
  • FGHM inline acceptance → FG qty added to stock.
  • Material Return → Qty added back to stock.
  • Min/Max stock → Alert on threshold cross (from Item Muster).

Conditional Visibility

  • GRN Transport subform visible if PO Scope of Transport = Own
  • GRN Client Site textbox visible if Warehouse = Client Site
  • Project Application tasks visible if Execution Base = Area Basis
  • Project Manpower visible if Execution Base = Day Basis

Partial GRN Logic

  • Checkbox per line item in GRN subform
  • Only checked items received; unchecked stay open on PO
  • GRN posting only adds checked items to stock

Zoho Books/CRM/Inventory Style Conventions

  • Like Zoho Books: PO header + N line items with proper GST split (CGST/SGST/IGST). Total in words. T&C block.
  • Like Zoho CRM: Project as the parent record. All sub-forms reference Project via lookup. Sales Order → Project auto-creation.
  • Like Zoho Inventory: Warehouse selection on GRN. Stock tracking by warehouse. Bin location mapping. Partial receipt.
  • Like Zoho Projects: Tasks by category. Conditional visibility per execution base. Budget tracking per task type.

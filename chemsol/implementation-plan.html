System Overview

Core principle: Everything starts from Project Acceptance. A Sales Order creates a Project. All procurement (PR, PO, GRN), inventory (MR, MIS, FGHM), and production flows are tagged to that Project. Nothing exists outside a project context.

Stock Management

  • GRN posted → Qty added to stock (per warehouse). Timestamp logged.
  • MIS posted → Qty deducted from stock.
  • FGAN approved → FG qty added to stock.
  • Material Return → Qty added back to stock.
  • Min/Max stock → Alert on threshold cross (from Item Muster).

Conditional Visibility

  • GRN Transport subform visible if PO Scope of Transport = Own
  • GRN Client Site textbox visible if Warehouse = Client Site
  • Project Application tasks visible if Execution Base = Area Basis
  • Project Manpower visible if Execution Base = Day Basis

Partial GRN Logic

  • Checkbox per line item in GRN subform
  • Only checked items received; unchecked stay open on PO
  • GRN posting only adds checked items to stock

Zoho Books/CRM/Inventory Style Conventions

  • Like Zoho Books: PO header + N line items with proper GST split (CGST/SGST/IGST). Total in words. T&C block.
  • Like Zoho CRM: Project as the parent record. All sub-forms reference Project via lookup. Sales Order → Project auto-creation.
  • Like Zoho Inventory: Warehouse selection on GRN. Stock tracking by warehouse. Bin location mapping. Partial receipt.
  • Like Zoho Projects: Tasks by category. Conditional visibility per execution base. Budget tracking per task type.

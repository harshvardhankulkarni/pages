Process of Productivity

Two-Stream Architecture: Stream A (Pre-Project): PURCHASE → STOCK (no Project ID). Stream B (Project-Centric): SALES → PROJECT → MR→MIS → PRODUCTION → COSTING → SERVICE → FINANCE → LOGISTICS (Project ID flows downstream). Procurement runs independently for stock. MR consumes from Store inventory for projects.

3-Layer Hierarchy: SO sells Systems (EP01) → composed of FGs (System Composition) → each FG formulated from RMs (BOM). MR works at RM level only.

PURCHASE→ STOCK ⇔ Stream A ➧ SALES→ PRODUCTION→ MR→MIS→ COSTING→ SERVICE→ FINANCE→ LOGISTICS

🔗 Backend Rule — Site Manager Consumption Entry → Project-Assigned RM

Front-end (what the Site Manager sees): The entry is made against the SO System (e.g., EP01) and the FG/work done — the system composition (System → FG → RM via BOM) is shown for reference.

Backend (what actually happens): The system expands the FG into its RMs using BOM ratios, then for each RM increments Consumed Qty on the matching line of the Project's MR Material Allocation (matched by Project ID + Item Code). It does NOT deduct from a generic stock pool for project-tagged material.

Site Consumption Entry (Project, System/FG, Qty) └─> expand via BOM: FG = RM₁×r₁ + RM₂×r₂ + ... └─> for each RMᵢ: MR.Allocation[Project, RMᵢ].ConsumedQty += Qty × rᵢ MR.Allocation[Project, RMᵢ].Consumption% = ConsumedQty / AssignedQty × 100 IF Consumption% ≥ 80% AND 80%AlertFlag → FIRE 80% ALERT └─> (same logic applies to FG production BMR / RM Consumption entries)

Reconciliation: MIS issued qty and FG-production consumption both roll up into the same Consumed Qty so the Project sees true remaining allocation at all times.

⚠ 80% Consumption Alert — Automation

Trigger: Per RM in the MR Material Allocation, when Consumed Qty ÷ Assigned Qty × 100 ≥ 80% and the 80% Alert Flag is ON.

Fires: (1) In-app pop-up to the entry user; (2) Dashboard banner on Project + Store + Service dashboards ("Material X at 80% of project allocation"); (3) Email to Project Manager & Coordinator. At 100% the banner escalates to "Allocation Exhausted".

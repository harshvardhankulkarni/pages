Implementation Reference

Automation Implementation Guide

Step-by-step instructions for implementing all automations in Zoho Creator. Covers Workflow Rules, Deluge Scripts, Blueprint/Approval Processes, Form Rules, Custom Functions, and Auto Numbering for every form in the system.

Zoho Creator Automation Types Quick Reference

Workflow Rule
Triggered on form Submit/Update/Delete. Use for: status changes, stock updates, send email, create child records. Runs server-side.
Deluge Script
Custom script for complex logic: loops, conditions, API calls, data manipulation. Embedded in Workflow Rules or Custom Functions.
Blueprint / Approval
Multi-step approval process. Use for: MR status workflow (Draft -> Production Verified -> Costing Approved -> Released). Supports conditional branching.
Form Rule
Client-side show/hide, validate, auto-populate. Use for: conditional subform visibility, field validation, dynamic dropdowns.
Custom Function
Reusable Deluge function. Call from Workflow Rules or Scheduler. Use for: stock recalculation, report generation.
Auto Numbering
Built-in field setting. Use for: sequential document numbers (PR-2026-0001, MR-2026-0001). Configure in field properties.

3. MR — Costing Gate CRITICAL AUTOMATIONS
Blueprint Workflow Deluge 3A · MR — Material Requisition [CRITICAL GATE]
MR STATUS WORKFLOW (Blueprint / Approval Process)

This is the most critical automation in the system. The MR status gates all downstream activity.

Draft→ Pending Verif→ Prod Verified→ Costing Approved→ Released
1
Create Blueprint: Zoho Creator -> Blueprint -> New Blueprint for MR form
2
States configuration:
BLUEPRINT: MR Status Workflow STATE 1: "Draft" (Initial - on create) -> Transition: "Submit for Production Verification" -> Action: Set Status = "Pending Production Verification" -> Condition: MR Line Items must have at least 1 item -> Notification: Send alert to Production dept: "New MR submitted for verification" STATE 2: "Pending Production Verification" -> Transition: "Verify (Production)" -> Action: Set Status = "Production Verified" -> Notification: Send alert to Costing dept: "MR verified by Production — pending Costing approval" -> Transition: "Send Back to Draft" -> Action: Set Status = "Draft" -> Reason required STATE 3: "Production Verified" -> Transition: "Approve (Costing)" -> Action: Set Status = "Costing Approved" -> Action: Calculate Total MR Cost (Material + Application + Transport + Tools) -> Notification: Send alert to Store + Production -> Transition: "Send Back to Draft" -> Action: Set Status = "Draft" -> Reason required STATE 4: "Costing Approved" -> Transition: "Release" -> Action: Set Status = "Released" -> Action: Lock all fields (read-only) -> Notification: Send email to Store + Production: "MR Released - proceed with MIS" STATE 5: "Released" (Terminal) -> No further transitions allowed -> MR is now available for MIS form lookup

Implementation Order Build Sequence

Build automations in this order (each depends on previous):

  1. Auto Numbering — Configure on ALL forms first (PR, PO, GRN, MR, MIS, FGHM, QC, SO, Project, BMR, Planning)
  2. Form Rules — Conditional visibility (SO Sales Type), lookup filters (MIS MR No), auto-populate (PO from PR, BMR RM list from BOM)
  3. Formula Fields — Total Amount on SO, GST split on PO, Total MR Cost, Allocation Ratio, Variance
  4. Blueprints — MR Status workflow (Draft -> Pending Prod Verification -> Prod Verified -> Costing Approved -> Released). This is the critical gate. Test thoroughly.
  5. Workflow Rules — SO -> Project creation, Stock updates (GRN -> RM+, MIS -> RM-, FGHM -> FG+), 80% alert, PR notifications
  6. Custom Buttons — "Post GRN", "Post MIS" with Deluge scripts for stock updates and logging
  7. Scheduled Workflows — Daily escalation reminders, Min/Max stock alerts
  8. Custom Functions — Reusable getCurrentStock(), stock movement summary functions
  9. Print Templates — PO printable format with GST split, company logo

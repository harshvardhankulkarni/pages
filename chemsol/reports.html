C Material Requisition (Cost Baseline) + Issue + FG Flow MR (Draft→Verified→Approved→Released) → MIS · FGHM
⛔ Critical Gate: MR is the complete project implementation cost baseline — four cost components (Material + Application + Transportation + Tools & Tackles) summing to the Total MR Cost that Costing approves. MR must be Released before MIS can issue stock. Status flow: Draft → Production Verified → Costing Approved (TOTAL cost) → Released. Without a Released MR, there is no MIS, no Production, no project execution.
  • MR (Project Cost Baseline — 4 components) Tagged to Project
    • Draft Initial state
      • Production Verified Qty vs SO checked
        • Costing Approved Approves TOTAL project cost
          • Released Gate passed
            • MIS Store issues — N items
              • Stock − Deducted
            • Production Tagged to Project
              • FGHM Handover + Inline Acceptance
                • Notify Store
                  • FG Stock + Updated

H Stock Management Rules
  • GRN posting — Qty added to stock only after posting; timestamp logged
  • MIS posting — Qty deducted from stock
  • FGHM inline acceptance — FG qty added to stock after inline acceptance
  • Material Return — Qty added back to stock
  • Min/Max stock — Alert when stock crosses thresholds
  • Consumption → Project-Assigned RM — Every consumption entry (BMR/RM Consumption) resolves to MR Allocation line matched by Project ID + Item Code
  • 80% Alert — When Consumption % ≥ 80% and Alert Flag = ON → pop-up + dashboard banner + email to Project Manager

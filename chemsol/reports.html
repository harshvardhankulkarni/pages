Deluge Automations — 24 Workflows

All automation is implemented using Zoho Deluge scripting across 6 categories:

Auto-Numbering (7)

PR-{000}, RM-{000}/RMWAD-{000}, GRN-{000}, MR-{000}, MIS-{000}, QC-{000}, SO-{000}/WO-{000}, PROJ-{000}, BOM-{000}

Auto-Fetch Lookups (10)

Item→Name/UOM/LeadTime, Supplier→GST/Bank, Customer→Contact/Address, System→Name/Thickness, GRN→PO items, QC→GRN items, MR→Available Stock

Calculations (4)

PO line: Amount=Qty×Rate, GST calc (CGST/SGST/IGST). SO/WO line: Amount=Area×Rate. Commission: % or Fixed.

Stock Updates (5)

GRN (+), MIS (−), Material Return (+), FG Receiving (+), Material Handover (transfer) — all trigger on Post.

Approvals (4)

PR (2-level), Rate Comparison, PO (2-level), MR (2-level) — with ₹50K/₹2L thresholds and Director escalation.

Validations (6+)

Duplicate Item Name, GST range (0-100%), Min≤Max Stock, HSN format (^\d{4,8}$), Insufficient stock on MIS, PO amendment after GRN.

Sample: PO GST Calculation

// GST Calculation per line item (On Qty/Rate/GST% change)
basicAmount = input.Qty * input.Rate;
gstAmount = basicAmount * input.GST_Percentage / 100;
if(input.GST_Type == "Intra-State") {
    input.CGST_Amount = gstAmount / 2;
    input.SGST_Amount = gstAmount / 2;
} else {
    input.IGST_Amount = gstAmount;
}
input.Total_Amount = basicAmount + gstAmount;

Sample: Post-GRN Stock Update

// Post GRN — Update Stock
for each item in selectedItems {
    itemRecord = zoho.creator.getRecordById("chemsol",
        "Item_Master", item.Item_Code);
    currentStock = itemRecord.get("Current_Stock");
    zoho.creator.updateRecord("chemsol", "Item_Master",
        item.Item_Code,
        {"Current_Stock": currentStock + item.Received_Qty});
}
input.Status = "Posted";
input.Posted_Timestamp = zoho.currenttime;

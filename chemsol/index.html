Architecture: Project Unites Everything

Just as Zoho CRM treats a Deal as the parent record with related lists (Quotes, Invoices, Contacts), Chemsol treats Project as the central hub. All transactions — PR, PO, GRN, MR, MIS, Production Orders, FG Handovers — carry a Project ID lookup. Nothing exists outside a project context.

Initial Process Flow

STEP 1: Sales Order (Supply+Apply) is accepted
STEP 2: Project record is auto-created with SO reference
STEP 3: Project Manager assigned → Execution begins
STEP 4: All procurement (PR→PO→GRN) tagged to this Project
STEP 5: Material requests (MR→MIS) tagged to this Project
STEP 6: Production planned & executed per Project
STEP 7: FG Handover & Acceptance per Project
STEP 8: 5 Task Categories track execution per Project

Procurement Flow (N Line Items)

Like Zoho Books Purchase Orders, each PR/PO/GRN has a line-item subtable with autofetch fields, GST split, and HSN codes.

Project→ PR→ Rate Comparison→ PO (RM/RMWAD)→ GRN (partial ✓)→ QC/QA→ Stock +

↳ Each PO line: Item Code, Name, HSN, Qty, UOM, Rate, Basic Amount, GST%, CGST/SGST/IGST, Total

Inventory Flow

Project→ MR→ MIS→ Stock −

Production → FG Flow

Project→ Production→ FGHM→ Notify Store→ FGAN→ FG Stock +

Departments

Purchase

PR, Rate Comparison, PO, GRN, Supplier management — all tagged to Project. Like Zoho Books purchase workflow.

Store & Logistics

GRN, MIS, Material Return, Handover, FG Receiving, Vehicle & Transport — per Project. Like Zoho Inventory warehouse ops.

Production & R&D

MR, BMR, Production Planning, RM Consumption, Packing, FGHM — per Project. Manufacturing execution.

Project

Create Project, 5 task categories, execution tracking. Like Zoho Projects — central to all operations.

Sales

SO/Work Orders, Customer/Site Master. Like Zoho CRM — SO acceptance triggers Project creation.

QC

QC/QA inspection, testing results on GRN items.

Account & Finance

Read-only on financial forms, project-wise payment tracking.

Admin

Full access, user management, approval matrix.

Core Philosophy

Every transaction originates from or is tagged to a Project. A Sales Order (Supply+Apply mode) auto-creates a Project. PRs, POs, GRNs, MRs, Production Orders, and FG movements all carry a Project ID lookup. Nothing exists outside a project context.

Core Flows (per IMPLEMENTATION_PLAN phase order)

SO (Supply+Apply)→ Project→ 5 Task Buckets
Project→ PR→ Rate Comparison→ PO (RM/RMWAD)→ GRN→ QC/QA→ Stock +
Project→ MR→ MIS→ Stock −
Project→ Production→ FGHM→ Notify Store→ FGAN→ FG Stock +

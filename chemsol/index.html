Core Process Flow

The end-to-end project lifecycle. MR acts as the critical costing approval gate — all downstream execution depends on MR being Released.

SO 📄 → Costing 💰 → MR 📋
Draft→Prod Verified→Costing Approved→Released → MIS 📝 → Production 🏭 → FGHM 📦 → FG Stock 📈
Procurement (PR→PO→GRN→QC) 🏗️ when stock insufficient
1 SO → Costing Review Sales Order creates Project. Costing team reviews RM + cost requirements.
→
2 MR — Draft Material Requisition with 4 cost components: Material, Application, Transportation, Tools.
→
3 Production Verified Production checks MR qty vs SO system req via BOM.
4 Costing Approved → Released ⛔ Costing approves TOTAL project cost. Critical gate passed — MIS can proceed.
→
5 MIS → Production Store issues RM via MIS. Production: Planning → BMR → RM Consumption → Packing.
→
6 FGHM → FG Stock FG Handover with inline acceptance. FG stock updated automatically.
S Side: Procurement (as needed) PR → PO → GRN → QC. Runs when inventory is insufficient. No Project ID.

FG from RM: How Finished Goods Are Created

The system traces Finished Goods back to their Raw Material composition. Every FG is defined by a Bill of Materials (BOM) which specifies which RMs make up that FG and in what ratio.

1 BOM Defines FG Composition BOM (FG Formulation) maps each FG → RM(s) with ratios. E.g., FG-EP-001 = 85% Epoxy Resin + 12% Hardener + 3% Pigment.
→
2 MR Allocates RM per Project Material Requisition records Assigned Qty per RM for the project (Material Allocation subform with ratio % and 80% alert flag).
→
3 MIS Issues RM from Store Material Issue Slip deducts RM from inventory. Only creatable after MR is Released. Tracks Issued Qty vs Required Qty.
4 BMR Consumes RM → Produces FG Batch Manufacturing Record consumes RM (deducting from MIS-issued stock) and outputs FG. Each entry resolves to the project-assigned RM on the MR Allocation line (Project ID + Item Code).
→
5 FGHM Hands Over FG FG Handover Master with inline acceptance. QC Pass/Fail/Hold check, damaged qty tracking, handover signatures.
→
6 FG Stock Increases On FGHM inline acceptance, FG quantity is added to FG Inventory. RM Inventory was already deducted at MIS posting.
Consumption tracking: BMR/RM Consumption entries increment Consumed Qty on the matching MR Allocation line (matched by Project ID + Item Code). At 80% consumption, a pop-up + dashboard banner + email alert fires to the Project Manager. At 100%, an "Allocation Exhausted" alert goes to Project Manager + Purchase.

Inventory Dual View: RM vs FG

Two distinct inventory pools tracked independently. RM Inventory is governed by procurement and material issuance; FG Inventory by production handover and sales dispatch.

Implementation Roadmap — 6 Phases

Phased rollout across all departments. Each phase corresponds to a vertical functional slice.

1 Master Data Foundation Item Muster · System · Supplier · BOM · Store

Global shared data — Item Muster (6 categories), System Master (EP/PU/DEM codes), System Composition (System→FG mapping), BOM Formulation (FG→RM with ratios), Supplier Master, Store Master (6 warehouses).

2 Sales & Project SO · Project · Task Budget

Sales Order with conditional subforms (System Lines / FG Lines), Project auto-created from SO (Supply+Apply mode).

3 Costing & MR (Critical Gate) MR · Costing Approval

MR with 4 cost components (Material, Application, Transportation, Tools), Material Allocation subform with 80% Alert, MR Status workflow: Draft → Pending Production Verification → Production Verified → Costing Approved → Released.

4 Procurement (As Needed) PR · PO · GRN · QC

Purchase Requisition, Purchase Order with dual numbering (RM/RMWAD) and GST split, Goods Receipt Note with partial checkbox, QC/QA linked to GRN.

5 Production & Inventory MIS · Planning · BMR · Pack · FGHM

MIS linked to MR (only Released MRs), Production Planning, BMR, RM Consumption, Packing Entry, FGHM with inline acceptance, RM + FG inventory management.

6 Reports & Dashboards Purchase · Store · Production · MR Status

Purchase Dept dashboard, Store Dept inventory dashboard, Production Dept dashboard, MR Status tracking report, 80% Consumption Alert dashboard, Project Material Allocation report.

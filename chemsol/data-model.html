Data Model & System Architecture

Two-stream architecture: Stream A (Pre-Project) — procurement for stock, no Project ID. Stream B (Project-Centric) — SO→Project, Project ID flows through MR→MIS→Production→Service→Finance→Logistics.

Flow 5 · Supplier Credit Note (Finance — Phase 8, Manually Initiated)

The Supplier Credit Note is always manually initiated by Account & Finance. It can originate from three trigger scenarios:

  • Trigger Events 3 Scenarios
    • Scenario 1: QC Rejected Items Defective
      • Finance manually creates Credit Note QC Reference
    • Scenario 2: GRN Short Delivery Shortage
      • Finance manually creates Credit Note GRN Reference
    • Scenario 3: Rate/Invoice Discrepancy Invoice
      • Finance manually creates Credit Note PO Reference
    • → Supplier Credit Note
      • Draft Finance Working
      • Approved Finance Senior
      • Credited Supplier Balance Updated
      • Cancelled No Impact

Design Decisions

DecisionChoiceRationale
ArchitectureProject-Centric HubAll modules tagged via Project ID lookup — like Zoho CRM Deals as parent
PO Line ItemsN items + GST splitLike Zoho Books — CGST/SGST/IGST at line level
Warehouse6 fixed + Client SiteLike Zoho Inventory warehouse tracking
Task BudgetSingle table with Category dropdownSimpler than 5 nested conditional sections
SO ModeSupply+Apply → ProjectEach SO creates a Project automatically
GRN PostingPost = stock update + timestampPrevents accidental stock updates
Partial GRNCheckbox per line itemReal-world partial deliveries
PO SeriesRMWAD (Coding) / RM (Non-Coding)Coding materials require stricter control
Bidirectional autofetchItem Code ↔ Item NameLike Zoho CRM lookup behavior
Supplier Credit — PO LinkMandatory PO ReferenceEvery credit note MUST reference the original PO for audit trail — like Zoho Books Credit Note linked to Purchase Invoice
Supplier Credit — SourceQC/QA inspection resultsAuto-fetch rejected qty from QC to minimize manual entry errors
Supplier Credit — StatusDraft → Approved → Credited → CancelledFinance workflow ensures proper approval before supplier balance adjustment

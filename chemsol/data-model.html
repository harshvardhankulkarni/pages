Flow 1 · Procurement (Phase 2)

Project (Root) └── PR (Production/R&D) — Project ID lookup └── Approval (Approval Matrix) └── Rate Comparison (Purchase — 5 suppliers) └── PO (Purchase — RM/RMWAD, GST split, N line items) └── GRN (Store — partial via checkbox) └── QC/QA (QC dept — test results) └── Stock Updated (after posting)

Flow 2 · Material Issue (Phase 3)

Project (Root) └── MR (Production/R&D) — Project ID lookup └── MIS (Store issues material — N items) └── Stock − (deduction on posting)

Flow 3 · FG Handover (Phase 4)

Project (Root) └── Production — Project ID lookup └── FGHM (FG Handover — N items) └── Pop-up Notification → Store/Logistics └── FGAN (FG Acceptance) └── FG Stock + (after approval)

Flow 4 · Sales → Project (Phases 5–6)

SO/Work Order (Dual Mode) ├── Supply+Apply → Auto-creates Project → 5 Task Categories │ ├── Transportation │ ├── Application/Execution (Area Basis) │ ├── Manpower (Day Basis) │ ├── Tools │ └── Additional Expenses │ └── Supply Only → Material Sales (FG dispatch — no project)

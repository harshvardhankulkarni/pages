Flow 1 · Procurement (Phase 2)

Project (Root)
  └── PR (Production/R&D) — tagged to Project
       └── Approval (Approval Matrix)
            └── Rate Comparison (Purchase)
                 └── PO (Purchase — RM/RMWAD series)
                      └── GRN (Store — partial via checkbox)
                           └── QC/QA (QC dept — test results)
                                └── Stock Updated (after posting)

Flow 2 · Material Issue (Phase 3)

Project (Root)
  └── MR (Production/R&D) — tagged to Project
       └── MIS (Store issues material)
            └── Stock − (deduction on posting)

Flow 3 · FG Handover (Phase 4)

Project (Root)
  └── Production — tagged to Project
       └── FGHM (FG Handover)
            └── Pop-up Notification → Store/Logistics
                 └── FGAN (FG Acceptance)
                      └── FG Stock + (after approval)

Flow 4 · Sales & Project (Phases 5–6)

SO/Work Order (Dual Mode)
  ├── Supply+Apply → Auto-creates Project → 5 Task Buckets
  │                                          ├── Transportation
  │                                          ├── Application/Execution (Area Basis)
  │                                          ├── Manpower (Day Basis)
  │                                          ├── Tools
  │                                          └── Additional Expenses
  │
  └── Supply Only → Material Sales (FG dispatch — no project)

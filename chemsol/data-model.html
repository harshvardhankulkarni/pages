Architecture · Relationships · Flow

Data Model & System Architecture

Two-stream architecture: Stream A (Pre-Project) — procurement for stock, no Project ID. Stream B (Project-Centric) — Project baseline = SO + MR (SO = revenue side, MR = complete project implementation cost baseline — total project cost: Material + Application + Transportation + Tools & Tackles — that Costing approves). SO→Project, Project ID flows through MR→MIS→Production→Service→Finance→Logistics. MR must be Production Verified → Costing Approved → Released before any downstream action.

Material Allocation & Consumption Tracking

MR = Complete Project Implementation Cost Baseline (four cost components): MR is NOT just RM allocation. It carries four cost components that sum to the Total MR Cost — the project implementation cost baseline that Costing approves:
  • Material Cost — from the per-project Material Allocation subform (one record per RM): Item Code, Item Name, UOM, Assigned Qty (allocation baseline), Rate, Material Cost (= Assigned Qty × Rate), Allocation Ratio % (share of total, sums to 100%), 80% Threshold Alert Flag (default ON), Consumed Qty (running total), Consumption % (= Consumed Qty ÷ Assigned Qty × 100), Alert Triggered (auto-set at 80%). This establishes the project's material consumption budget.
  • Application Cost — labour/execution cost to apply material on site (Activity, UOM, Qty/Area, Rate, Amount).
  • Transportation Cost — transport of material to site (From, To, Vehicle/Trips, Rate, Amount).
  • Tools & Tackles — tools/equipment needed (Item, Qty, Rate, Amount).
Total MR Cost = Material Cost + Application Cost + Transportation Cost + Tools & Tackles. Costing approves the TOTAL cost — not just the material portion.
Project Profitability Summary — for Sales / Actual Team: Costing must give the sales/actual team a clear profitability view: SO Revenue − Total Cost = Gross Margin / Margin %.
  • SO Revenue — from SO System Lines (Σ Area × Rate) + SO Transportation.
  • Total Cost = Material Cost + Application Cost + Transportation Cost + Tools & Tackles (the four MR cost components) + Production Cost (BMR RM Consumption + Packing).
  • Gross Margin = Revenue − Total Cost; Margin % = (Gross Margin / Revenue) × 100.
If Margin % falls below the set threshold, Costing flags the project before approval.
MR Status Workflow (Critical Gate): The MR carries a MR Status field with four states: Draft (initial) → Production Verified (Production checks MR quantities against SO system requirements via BOM) → Costing Approved (Costing approves the TOTAL project cost — all four components) → Released (MR is actionable). Without Released status, MIS cannot issue stock, Production cannot start, and Logistics cannot dispatch.
Consumption Resolution Rule: Both FG production (BMR / RM Consumption) and Site Manager consumption entries increment Consumed Qty on the MR Allocation line matched by Project ID + Item Code — they never deduct from a generic stock pool for project-tagged material. Site Manager entries against the SO System/FG are first expanded into RMs at BOM ratios (System Composition → BOM), then each increment is applied to the matching Allocation line.
80% Consumption Alert: When any allocated RM's Consumption % ≥ 80% and the 80% Threshold Alert Flag is ON, the system fires three simultaneous notifications:
  • Pop-up to the user saving the consumption entry
  • Dashboard banner on the Project dashboard
  • Email to Project Manager + Coordinator
At 100% consumption, the status escalates to "Allocation Exhausted" with a secondary alert.

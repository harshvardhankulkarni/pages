User Guide Project Budget Tracking

Module-by-module instructions for using the Zoho Creator application — what to enter, when, and how records connect.

This guide follows natural data entry order, not build order. Start at Vendors and work down.
Vendors→ Accounts→ Warehouses→ Items→ Projects→ Budget→ Stock→ PR→ PO→ GRN→ SCN→ Bill→ Payment→ Exp→ BOM→ DC→ Inv→ Reports
Select a module from the sidebar to get started.

STEP 7 Inventory Transactions Inventory_Transactions

▷

Overview

▸

Every stock movement — receiving, consuming, adjusting, reserving — is an Inventory Transaction. One entry triggers stock updates and budget tracking.

Highest-value automation: Stock Out + Project selection auto-creates an Expense record.
▷

Transaction Types

▸
TypeWhen to useStock effect
Stock InReceiving goods from supplierCurrent_Stock += Qty
Stock OutConsuming materials for a projectCurrent_Stock −= Qty; auto-creates Expense
Stock AdjustmentCorrecting physical countCurrent_Stock = Adjusted_Qty
Return to VendorSending defective goods backCurrent_Stock −= Qty
ReservationReserving stock for a projectReserved_Qty += Qty
ReleaseReleasing reserved stockReserved_Qty −= Qty

Key Fields

▸

Select the item being moved.
Warehouse Lookup → Warehouses
Which warehouse's stock changes.
Quantity Decimal
Always positive. Direction determined by Transaction Type.
Project Lookup → Projects
Required for Stock Out and Reservation.
Rate / Unit Cost Currency
Cost per unit at transaction time.
▷

Validation

▸
Stock Out blocked if quantity exceeds (Current_Stock − Reserved_Qty). Check stock or release reservations first.

Setup Notes

▸

Deluge: On Submit — for each transaction type: Stock In increases warehouse stock, Stock Out decreases it (with negative stock prevention), Reservation increments Reserved_Qty. All changes update Item_Warehouse_Stock then roll up to Item.Current_Stock.

Auto-Expense: When Stock Out has a Project set, Deluge auto-creates an Expense record (Qty × Rate) for project cost tracking.

Vendor Bills (AP Sub-ledger) Vendor_Bills

Records invoices received from vendors against Purchase Orders. This is the Accounts Payable sub-ledger — the finance team records the bill, performs 3-way matching (PO × GRN × Bill), and tracks payment status through to settlement. Part of Phase 1G.

How Vendor Bills Fit in the Procurement Cycle

PO→ Goods Receipt→ Supplier Credit Note→ Vendor Bill→ Payment

Bill DateDateDefault today
Due DateDatePayment due date
Reference NumberTextVendor's invoice number

Balance DueCurrency (Formula)Total_Amount − Amount_Paid
StatusDropdownDraft → Received → Matched → Approved → Partially Paid → Paid → Cancelled

3-Way Match Process

When Status changes to "Matched", Deluge validates:

  • PO Ordered Qty ≥ Bill Qty — cannot bill more than ordered
  • GRN Accepted Qty ≥ Bill Qty — cannot bill more than received
  • If either check fails, an alert is shown and status stays "Received"
  • If both pass, status auto-advances to "Approved"

Key Automations

#29Bill Received → record vendor invoice, notify finance for 3-way match
#30Bill Matched → 3-way match validation (PO × GRN × Bill), calculate PPV
#31Bill Approved → finalize PPV, update AP metrics
#32Bill Cancelled → validate no linked Payments or SCNs

Reports

  • AP Aging — Bills grouped by Due Date range
  • Outstanding Bills — Unpaid bills
  • PPV Register — Purchase Price Variance by line item
  • Vendor Payment History — All bills and payments per vendor

Currency Exchange Rates Currency_Rates

Manages foreign exchange rates for multi-currency transactions. FX rates are auto-populated on Bills, Invoices, and Payments based on the transaction date. Part of Phase 1H.

How It Works

When a Bill or Invoice is created in a foreign currency, Deluge looks up the nearest prior FX rate from Currency_Rates matching the transaction date. If no rate is found, the user can enter it manually.

Immutable audit trail for all financial transactions. Every status change, amount edit, and record creation across all P0 financial forms is automatically logged. Write-only via Deluge — users cannot create or edit audit records directly. Part of Phase 1I.

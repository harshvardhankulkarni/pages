How to Use This Guide

This guide follows the natural data entry order. Start at the top and work down. Each module section shows:

When to create → Key fields → How it connects → What happens automatically
Building order (Phase 1A → 1F) is different from data entry order. You build modules top-down, but enter data in this sequence.
STEP 1

Vendors

Vendors
Vendors stores your suppliers — companies that sell you materials, equipment, and services. Create vendors first before creating POs or recording expenses.
Data entry rule: Create each vendor here before creating a PO or recording an expense against them.

Warehouses

STEP 4

Inventory Items

Inventory_Items
Items are everything you buy, sell, stock, or consume — raw materials, components, finished goods, and services. Every item gets a unique SKU. Stock is tracked automatically across warehouses.
Goods vs Services: Set Item Type = "Goods" for physical items that need stock tracking. Set "Services" for non-stock items (labour, consulting). Only Goods get warehouse stock records.
Inventory_Items ──┬── Item_Warehouse_Stock (embedded subform: stock per warehouse)

Required

  • Item Name

Important for POs

  • Purchase Price (default unit cost)
  • Tax Rate (default for PO lines)
  • Preferred Vendor (auto-fills on POs)
  • HSN/SAC Code (tax classification)

Stock tracking — how it works

Auto When you create a Goods item, the system auto-creates stock records for all active warehouses with 0 quantity.
Auto Every Inventory Transaction (Stock In/Out) updates Item_Warehouse_Stock.Current_Stock and rolls up to the Item's Current_Stock total.

STEP 5

Projects — the central hub

Projects
Projects are the core of the system. Everything — budgets, expenses, inventory consumption, POs, invoices — ties back to a project. Create a project for each client engagement, internal initiative, or manufacturing job.
Almost every other module has a "Project" lookup field. Always select the right project so P&L reports are accurate.

Required

  • Project Name

Auto-generated

  • Project Code (PROJ-0001)

The person responsible for this project.
Total Approved Budget Currency
Set the top-line budget. Later, Budget Plans use this as the ceiling.
Status Dropdown
Planning → Active → On Hold → Completed → Closed. When set to Completed, system checks for open expenses/POs.

Custom Buttons on the Project form

STEP 6

Budget Plans & Components

Budget_Plans + Budget_Components
Budget Plans define how much money is allocated to a project. Budget Components break that into categories (Labour, Materials, Travel, etc.). Expenses check against these budgets automatically.
One active plan per project: The system prevents creating a second active budget plan for the same project. Close or revise the existing plan first.
Budget_Plans ── Budget_Components (embedded subform)
│ Component Name: "Materials" — Allocated: 5,00,000
│ Component Name: "Labour" — Allocated: 3,00,000
│ Component Name: "Travel" — Allocated: 50,000
│ Sum of all components ≤ Plan's Total Budget Amount
Budget Plan — Project Lookup → Projects
Select the project. One active plan per project enforced by Deluge.
Budget Components — Allocated Amount Currency
Set how much for each category. The sum must not exceed the plan's Total Budget Amount.
Status Dropdown
Budget Component status auto-updates as expenses hit it: Within Budget → 80% Alert → 90% Alert → Exceeded.
Referenced by →
Expenses
Expense.Budget_Component lookup
Referenced by →
Budget_Approvals
Created on overrun

How budget alerts work

AutoExpense submitted → system checks if new total exceeds 80%/90%/100% of Allocated Amount → auto-updates status + sends alert
STEP 7

Inventory Transactions

Inventory_Transactions
Every stock movement — receiving goods, consuming materials, adjusting counts, reserving for projects — is recorded as an Inventory Transaction. This is the most automated module: one entry triggers stock updates, expense creation, and budget tracking.
Highest-value automation: When you do a "Stock Out" and select a Project, the system auto-creates an Expense record. This closes the loop between inventory consumption and budget tracking.

Transaction types you can create

TypeWhen to useStock effect
Stock InReceiving goods from supplier (or use GRN for auto-creation)Item_Warehouse_Stock += Qty
Stock OutConsuming materials for a projectItem_Warehouse_Stock −= Qty; auto-creates Expense
Stock AdjustmentCorrecting physical count vs system countCurrent_Stock = Adjusted_Qty
Return to VendorSending defective goods backItem_Warehouse_Stock −= Qty
ReservationReserving stock for a specific projectReserved_Qty += Qty
ReleaseReleasing reserved stock back to poolReserved_Qty −= Qty
Item Lookup → Inventory_Items
Select the item being moved.
Warehouse Lookup → Warehouses
Which warehouse's stock changes.
Quantity Decimal
Always enter as a positive number. The Transaction Type determines direction.
Project Lookup → Projects
Required for Stock Out (consumption) and Reservation. Select the project this stock is used for.
Rate / Unit Cost Currency
Cost per unit at time of transaction. Used for auto-expense calculation.
Validation: System prevents Stock Out if quantity exceeds available stock (Current_Stock − Reserved_Qty). If you see an error, check stock or release reservations first.

Only create GRN against an existing PO. Select the PO → vendor, items, and quantities auto-fill from the PO line items.

STEP 12

Expenses

Expenses
Expenses track every rupee spent on a project — vendor bills, employee reimbursements, material costs. Expenses can be entered manually OR created automatically when you do a Stock Out transaction.
Budget check happens on submit: If the expense + existing spending exceeds the Budget Component's Allocated Amount, status goes to "Overrun-Pending Approval" and a Budget_Approval record is created.
Project Lookup → Projects
Select the project this expense belongs to.
Budget Component Lookup → Budget_Components
Filtered by selected Project. Choose "Materials", "Labour", etc.
Amount Currency
The expense value.
Vendor Lookup → Vendors
Optional — who was paid.
Supporting Document
Upload invoice or receipt.
Status
Draft → Submitted → Approved / Overrun-Pending Approval / Rejected.

Auto-created expenses (from Stock Out)

Auto Stock Out transaction + Project → system creates Expense:
Amount = Qty × Rate, Description = "Auto: Stock consumed — [Item] for [Project]", Status = "Approved"

Budget_Approvals
Created automatically when an expense exceeds the budget. PM and Finance Manager review, approve (which adjusts the budget), or reject.
You don't create this manually. It's auto-generated when an Expense goes over budget.

Approval Status

Modified Budget Amount

AutoRejected → Expense status = "Rejected", notifies requester

End-to-End Data Flow

How a typical procurement-to-payment journey flows through the system:

Procurement Flow:
PR Created → 3-Stage Approval → PO Auto-Created → PO Sent to Vendor → Goods Received (GRN) → Stock Updated → PO Auto-Closed

Project Consumption Flow:
Stock Out Transaction → Expense Auto-Created → Budget Check Runs → (if over) Budget Approval Created

Revenue Flow:
Invoice Created → Invoice Sent → Payment Received → Project P&L Updated

P&L Calculation (per project):
Profit/Loss = Total Invoiced − Total Expenses − Stock Consumed (auto-expenses)

User Guide Project Budget Tracking

Module-by-module instructions for using the Zoho Creator application — what to enter, when, and how records connect.

This guide follows natural data entry order, not build order. Start at Vendors and work down.
Vendors→ Accounts→ Warehouses→ Items→ Projects→ Budget→ Stock→ PR→ PO→ GRN→ Exp→ BOM→ DC→ Inv→ Reports
Select a module from the sidebar to get started.

STEP 6 Budget Plans Budget_Plans + Budget_Components

▷

Overview

▸

Budget Plans allocate money to a project. Budget Components break that into categories (Labour, Materials, Travel, etc.). Expenses check against these budgets automatically.

One active plan per project: System prevents creating a second active budget for the same project.
Budget_Plans ── Budget_Components (embedded subform)
Component Name: "Materials" — Allocated: 5,00,000
Component Name: "Labour" — Allocated: 3,00,000
Sum of all components ≤ Plan's Total Budget Amount

Key Fields

▸

Total Budget Amount Currency
Top-line allocation.
Budget Components — Allocated Amount Currency
Per-category allocation. Sum must not exceed Total Budget.
Status
Draft → Active → Closed
▷

Budget Alerts

▸

AutoOverrun triggers Budget_Approval record creation
▷

Relationships

▸
Referenced by →
Expenses

Referenced by →

STEP 7 Inventory Transactions Inventory_Transactions

▷

Overview

▸

Every stock movement — receiving, consuming, adjusting, reserving — is an Inventory Transaction. One entry triggers stock updates and budget tracking.

Highest-value automation: Stock Out + Project selection auto-creates an Expense record.
▷

Transaction Types

▸
TypeWhen to useStock effect
Stock InReceiving goods from supplierCurrent_Stock += Qty
Stock OutConsuming materials for a projectCurrent_Stock −= Qty; auto-creates Expense
Stock AdjustmentCorrecting physical countCurrent_Stock = Adjusted_Qty
Return to VendorSending defective goods backCurrent_Stock −= Qty
ReservationReserving stock for a projectReserved_Qty += Qty
ReleaseReleasing reserved stockReserved_Qty −= Qty
▷

Key Fields

▸
Item Lookup → Inventory_Items
Select the item being moved.
Warehouse Lookup → Warehouses
Which warehouse's stock changes.
Quantity Decimal
Always positive. Direction determined by Transaction Type.
Project Lookup → Projects
Required for Stock Out and Reservation.
Rate / Unit Cost Currency
Cost per unit at transaction time.
▷

Validation

▸
Stock Out blocked if quantity exceeds (Current_Stock − Reserved_Qty). Check stock or release reservations first.
Warehouse required for all types. To_Warehouse required for Transfer.
▷

Setup Notes

▸

Deluge: On Submit — for each transaction type: Stock In increases warehouse stock, Stock Out decreases it (with negative stock prevention), Reservation increments Reserved_Qty. All changes update Item_Warehouse_Stock then roll up to Item.Current_Stock.

Auto-Expense: When Stock Out has a Project set, Deluge auto-creates an Expense record (Qty × Rate) for project cost tracking.

STEP 8 Expenses Expenses

▷

Overview

▸

Every rupee spent on a project — vendor bills, reimbursements, material costs. Can be manual or auto-created from Stock Out transactions.

Budget check on submit: If expense exceeds component budget, status = "Overrun-Pending Approval" and Budget_Approval auto-created.
▷

Key Fields

▸
Project Lookup → Projects
Which project this expense belongs to.
Budget Component Lookup → Budget_Components
Filtered by selected Project.
Amount Currency
Must be > 0.
Vendor Lookup → Vendors
Optional — who was paid.
Status
Draft → Submitted → Approved / Overrun-Pending Approval / Rejected
▷

Workflow

▸
Draft→ Submitted→ Approved| Overrun-Pending→ Approved/ Rejected
▷

Setup Notes

▸

Deluge: On Submit — updates Budget_Component.Total_Spent, recalculates utilization %, triggers email alerts at 80%, 90%, and 100% consumption. If expense exceeds component budget, creates Budget_Approval record and notifies PM + Finance Manager.

└── Cancelled (only if no GRN linked)

PO Line Items (embedded subform)

▸
FieldAuto-fillNotes
ItemSKU, Name, Unit, HSNSelect → most fields pre-fill
Quantity / Unit RatePurchase PriceAdjust if needed
Discount % / Amount—Line-level discount
Tax %Tax Rate from ItemDefaults from item master
Item TotalFormula(Qty × Rate) + Tax − Discount

Setup Notes

▸

Deluge: On Status=Open — emails PO PDF to vendor. PO lifecycle: Draft → Open → Partially Invoiced → Invoiced → Closed / Cancelled.

Manual Close: POs are closed manually only — no auto-close based on GRN. GRN creates Stock In but does not change PO status.

Validation: PO cannot be Cancelled if linked GRN exists.

End-to-End Data Flow

▷

Procurement Flow

▸
PR Created → 3-Stage Approval → PO Auto-Created → PO Sent → GRN Created → Stock Updated → PO Closed
▷

Project Consumption Flow

▸
Stock Out → Expense Auto-Created → Budget Check → (if over) Budget Approval
▷

Revenue Flow

▸
Invoice Created → Invoice Sent → Payment Received → Project P&L Updated
▷

P&L Calculation

▸
Profit/Loss = Total Invoiced − Total Expenses

Overview handoff

This document is the formal handoff from design to implementation. It specifies every form, field, lookup, Deluge workflow, validation rule, report, and role permission needed to build the system in Zoho Creator.

Who should read this: Zoho Creator developer / implementation partner building the application in the Zoho Creator console.

Key design principle: All modules live in a single Zoho Creator application. Phase 1 handles everything in-platform. Phase 2 adds Zoho Inventory / Books / Projects API integration. The data model is designed to make migration between phases a copy operation, not a redesign.
Single most important performance decision: Maintain Item_Warehouse_Stock.Current_Stock via Deluge on every transaction — never aggregate stock on demand across thousands of rows.

Edge Cases & Failure States developer notes

Negative stock: Deluge must validate Item_Warehouse_Stock.Current_Stock >= Quantity before any Stock Out. Reject the transaction with a user-visible error if insufficient stock.
Partial Deluge failure: If a multi-step workflow (e.g. create Stock Out + create Expense) fails mid-way, partial data persists. Add safety checks: verify state before each write. Log failures to a Deluge_Error_Log form (text fields for Form Name, Record ID, Error Message, Timestamp).
Concurrent expense submissions: Two users submitting expenses against the same Budget Component simultaneously could both pass the budget check if it runs at the same moment. Zoho Creator's record-level locking mitigates this, but add a retry pattern: re-read Total_Spent inside the workflow, re-check, and fail with "Budget exceeded by another transaction — please resubmit" if needed.
Stale lookup values: Lookup fields cache the value at the time the record was created. For live stock counts, use a Formula Lookup field (recalculated on every form load) instead of a regular Lookup.
Empty states: Every report should show a friendly "No data yet" message when the source module has zero records. Dashboard KPI cards should show ₹0.00 / 0% / 0 instead of blank.
Deleting referenced records: Zoho Creator prevents deleting a record that is referenced by a Lookup in another form. Plan for archiving (Status = "Inactive") instead of deletion for Projects, Vendors, Items, and Warehouses.
